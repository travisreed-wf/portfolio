Kinect Gamepack

My freshman year at Iowa State I worked with four other CPrE majors to build an application called the Kinect GamePack. It allows up to two players to play tic-tac-toe together using body movements to select squares, and also includes poker with a single human player against three computer players. Each of the computer players determined a strength of hand based upon the odds of them successfully aquiring various hands, then modified that number by considering chip lead, table strength, and betting history. Once this final number was calculated we used a random number generator and the strengh to determine if it should call, raise or fold. We all worked on the initial Kinect integration together, then my primary responsiblility for the rest of the project was to design the AI.

This project taught me a lot about build algorithms that can take into account many variables, and it was an interesting way to learn how to interpret and use an unfamiliar interface.

Simulated Vending Machine

My fresman year I built a simulated vending machine using an Altera Cyclone II board. This "vending machine" allowed users to "insert" quarters, dimes, or nickels using the coresponding buttons on the board, then select an item to be vended using the switches. Throughout this process, the 7-segment display was continually updated with the money left.

This project taught me the value of verilog. First, I tried to complete the circuit by using a circuit diagram and a lot of wires. However, debugging this mess of wires was awful. However, when I tried it again using verilog it was much more simple. I also learned the value of splitting up projects into small, easily testable modules.
